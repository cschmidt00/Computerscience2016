What is Vindinium?

Vindinium is a game where players code their bots to collect mines, fight other players to steal their mines,
and collect taverns to gather health.The way to win a game is to have the most gold at the end of the turns, to do
this you need to have the most amount of mines. you respawn instantly, but the person who killed you gets all of your mines!

What is AI?

AI stands for Artificial Intelligence, meaning it has some logic or decision making.
What this looks like when it comes to coding is the bot has conditions where it will decide what is the best thing to do.
An example of this is where my code looks like the following:

if (bot.yourBot.life <=70 & bot.yourBot.gold> 3) {
task = "tavern";
}

This code is checking for when my bots health is less than 70 and aslong as I have more than 3 gold it will set
my task to tavern which later my bot checks what the task is set to and acts based on what its task is set to.

How does my bot work?

There are three componenets to my bot, there is What my bot is going to do, how its going to do it, and global data.

What To Do

This image includes the code that determines what to do, the way it works is the code checks to see if my bots health is equal to or less than 70 and my bot has more than 3 gold, it will set my task to tavern. if I have more mines than the leader and my health is less than 90 It also sets the task to tavern. This basically means if I have the most amount of mines it will make my bot play more defensively. The next decision my bot can make is to kill the leader, if the leader has more mines than me and also has more than 1 (so my bot doesnt kill people at the start who have 1 mine) it will set my task to killhim. At the start of the game before my bot loses health and before there is a leader to kill, my bot checks to see if my minecount is more than half the freemines left avaliable and make sure I have enough health to gather mines, more than 70. if I have enough health and there are enough free mines, my code will set the task to freemines the last thing my bot can do is steal a mine, this checks if there is an owned mine within 6 units and if there is than my bots task is set to steal

Global Data

This image shows off the global data that is essential for the code to know how to complete the tasks it is asigned. At the top there is a variable called "myDir" which is used later on to tell the bot which direction to walk in. the variable "myPos" is used to keep track of wherever my bot is whenever I need that informaiton. The variable "enemyMines" creates an array of the mines that are owned by the enemy bots inorder to find the closest enemy owned mine later on. The variable "leader" cycles through the enemyBots array and checks to see if the enemyBot minecount that it is currently checking is greater than the leaders minecount and if it is, it sets that bot as the leader. The variable called "ownedMines" cycles through the enemyMines array and checks the distance between myPos and an enemy mine and checks all of the others comparing which one is closer and sets that to ownedMines closestMine checks all of the freemines and find the closest one.
